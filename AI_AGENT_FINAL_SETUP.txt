STEP 10: APPROVE/REJECT RECOMMENDATIONS
================================================================================

Option A: Via Telegram (Interactive - COMING SOON)
---------------------------------------------------
Note: Buttons are shown but webhook isn't set up yet.
For now, use Option B below.

Option B: Via Supabase (Works Now)
-----------------------------------
1. Go to your Supabase dashboard:
   https://supabase.com/dashboard

2. Select your project

3. Click "SQL Editor" in left sidebar

4. Run this query to see pending recommendations:

   SELECT
     id,
     action_type,
     details->>'action' as action,
     details->>'reasoning' as reasoning,
     details->>'expected_impact' as impact,
     details->>'priority' as priority,
     ai_confidence
   FROM google_ads_agent_actions
   WHERE requires_approval = true
     AND approved_at IS NULL
     AND rejected_at IS NULL
   ORDER BY
     CASE details->>'priority'
       WHEN 'high' THEN 1
       WHEN 'medium' THEN 2
       WHEN 'low' THEN 3
     END;

5. To APPROVE a recommendation (replace XXX with actual id):

   UPDATE google_ads_agent_actions
   SET
     approved_at = NOW(),
     approved_by = 'manual-approval'
   WHERE id = XXX;

6. To REJECT a recommendation:

   UPDATE google_ads_agent_actions
   SET
     rejected_at = NOW(),
     rejected_by = 'manual-rejection',
     rejection_reason = 'Too aggressive for now'
   WHERE id = XXX;

7. To approve ALL high priority recommendations:

   UPDATE google_ads_agent_actions
   SET
     approved_at = NOW(),
     approved_by = 'batch-approval'
   WHERE requires_approval = true
     AND approved_at IS NULL
     AND rejected_at IS NULL
     AND details->>'priority' = 'high';

================================================================================
TROUBLESHOOTING
================================================================================

Error: "ANTHROPIC_API_KEY not found"
- Check .env file has the key
- Key should start with sk-ant-api03-
- No spaces around the = sign
- File must be in project root

Error: "TELEGRAM_BOT_TOKEN not found"
- Check .env file
- Token should look like: 7123456789:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw
- Make sure you started the bot in Telegram (Step 2.3)

Error: "Failed to send Telegram message"
- Did you click START on your bot in Telegram? (Step 2.3)
- Check chat ID is correct (your user ID from @userinfobot)
- Check bot token is correct

AI returns no recommendations:
- This can happen if no optimizations are needed (good sign!)
- Check the executive summary for insights
- Analysis still saved to database

Claude API rate limit:
- Rare on paid plan
- Wait 5 minutes and try again
- Consider upgrading API tier if frequent

GitHub Actions fails:
- Check GitHub Secrets are added correctly
- Check workflow file has new environment variables
- Look at the error logs in GitHub Actions tab
