# EduCourse Landing Page CRO Analysis & Recommendations

**Date:** March 28, 2026
**Page Analyzed:** Homepage/Landing Page (Landing.tsx)
**Primary Goal:** Sign-ups for test preparation courses ($199/year)
**Traffic Context:** Likely mix of organic search (parents searching for test prep), Google Ads, and direct

---

## Executive Summary

Your landing page has **strong fundamentals** but is leaving conversions on the table. The biggest opportunities are:

1. **Headline lacks specificity** - "Australia's Leading Test Preparation Platform" doesn't communicate unique value
2. **CTA copy is weak** - "Find Your Test" doesn't convey value or urgency
3. **Price positioning needs work** - $199 appears late and without proper anchoring
4. **Social proof is vague** - "1000+ families" lacks credibility signals
5. **Multiple friction points** - especially in the product selection flow

**Estimated Impact:** These changes could improve conversion by 30-50% based on industry benchmarks.

---

## 1. Value Proposition & Headline Issues (CRITICAL)

### Current State
**Headline:** "Australia's Leading Test Preparation Platform"
**Subheadline:** Long paragraph mentioning NAPLAN, selective entry, scholarship exams

### Problems
- ❌ "Leading" is a claim without proof - low trust
- ❌ Doesn't answer "what will I get?" or "why should I care?"
- ❌ Not specific enough to be meaningful
- ❌ Doesn't differentiate from competitors
- ❌ Subheadline is too long (3 lines) - visitors won't read it

### Recommended Changes

**Option 1 (Outcome-focused):**
```
Get Your Child Into Their Top-Choice School
Access 2,200+ practice questions, 5 full tests, and instant AI feedback
for NAPLAN, selective entry, and scholarship exams
```

**Option 2 (Problem-solution):**
```
Stop Wasting $800+ on Tutors. Get Better Results for $199.
Everything your child needs to ace NAPLAN, selective entry,
and scholarship tests - in one platform
```

**Option 3 (Specificity wins):**
```
James Ruse. Melbourne High. ACER Scholarships.
Join 1,000+ Australian families preparing with expert-designed
test prep for $199/year (vs $800-1,500 for tutoring)
```

**Why This Works:**
- Specific outcome (school admission)
- Concrete numbers create credibility
- Price comparison creates anchoring
- Target schools create aspiration

## 2. Call-to-Action Optimization (HIGH IMPACT)

### Current CTAs - WEAK
1. "Find Your Test" - implies browsing, not buying
2. "See how it works" - passive, exploratory
3. "See All Test Packages" - catalog browsing
4. "Get Started Today" - generic, no value

### Problems
- ❌ No urgency
- ❌ No value communication
- ❌ Implies research mode, not purchase mode
- ❌ "Find" suggests they might not find the right one

### Recommended CTAs

**Primary CTA (Hero):**
```
Get Started - 7-Day Money-Back Guarantee
```
*Why: Removes risk, implies purchase, clear action*

**Secondary CTA (Hero):**
```
See Pricing & Packages
```
*Why: Honest, direct, shows confidence*

**Problem Section CTA:**
```
Start Unlimited Practice for $199
```
*Why: Price + value + action*

**Methodology Section CTA:**
```
Try Risk-Free for 7 Days
```
*Why: Reduces friction, emphasizes guarantee*

**Footer CTA:**
```
Join 1,000+ Families - Start Today
```
*Why: Social proof + action*

### Button Hierarchy Issues
- ✅ Good: Primary buttons are visually prominent
- ❌ Bad: Too many CTAs competing for attention (5+ different actions)
- ❌ Bad: "Login" button in nav competes with "Get Started"

**Recommendation:** Remove "Login" from navigation for non-logged-in users, or make it text-only link in footer.

## 6. Friction Points & Obstacles

### Form Friction
- ❌ Unknown: Can't see checkout form, but ensure:
  - Maximum 5 fields (Name, Email, Password, Payment)
  - No "Phone Number" requirement
  - Clear password requirements
  - Guest checkout option

### Navigation Friction
- ❌ Desktop dropdown menu requires hover - not obvious it's clickable
- ❌ Mobile menu requires click to expand - consider mega menu
- ❌ "Insights" link in nav goes to external subdomain - confusion

### Content Friction
- ❌ Methodology section requires scrolling through 3 large screenshots - too much
- ❌ FAQ accordion requires clicks to read - friction
- ❌ School logos scroll automatically - users can't interact

### Loading Friction
- ⚠️ Multiple large screenshots (400x240px each) - optimize these
- ⚠️ GIF demo - likely heavy file size
- ⚠️ Framer Motion animations - could delay interaction

**Recommendations:**
1. Lazy load images below fold
2. Compress all images (WebP format)
3. Reduce animation complexity
4. Make first 3 FAQ items open by default
