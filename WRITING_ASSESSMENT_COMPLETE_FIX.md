# Writing Assessment - Complete End-to-End Fix for questions_v2

## Problem Summary
The writing assessment system was designed for the old `questions` table (v1) but needs to work with `questions_v2`. The foreign key constraint on `writing_assessments.question_id` prevents storing assessments for dynamically generated writing questions.

## Complete Solution

### Issue 1: Foreign Key Constraint Blocking Assessment Storage
**Problem:** `writing_assessments` table has `question_id UUID REFERENCES questions(id) NOT NULL`
**Impact:** 409 Conflict error when trying to store assessments for questions not in the database

### Issue 2: questions_v2 Schema Differences
**Problem:** Service tries to SELECT `writing_prompt` column that doesn't exist in `questions_v2`
**Impact:** 400 Bad Request error

### Issue 3: Writing Questions Not Stored in Database
**Problem:** Writing questions are dynamically generated per session but not persisted to `questions_v2`
**Impact:** Assessment service can't fetch question data from database

## Complete Fix Steps

### Step 1: Apply Database Migration

Run this SQL in your Supabase SQL Editor (or via migration):

\`\`\`sql
-- Remove foreign key constraint to allow dynamic question IDs
ALTER TABLE writing_assessments
  DROP CONSTRAINT IF EXISTS writing_assessments_question_id_fkey;

-- Add comment explaining the change
COMMENT ON COLUMN writing_assessments.question_id IS
  'UUID of the question - may reference questions, questions_v2, or be a dynamically generated question ID';
\`\`\`

**Why:** This allows storing assessments for any question ID, including dynamically generated ones.

### Step 2: Code Changes (Already Applied)

1. **writingAssessmentService.ts** - Now accepts question data directly from session
2. **TestTaking.tsx** - Passes question.text and question.subSkill from session
3. **Error handling** - Gracefully handles foreign key errors as warnings

### Step 3: Generate and Store Writing Questions in questions_v2

Writing questions should be generated and stored in `questions_v2` for each practice test. Currently they're only in session memory.

**Action Required:** Ensure writing question generation scripts store questions in `questions_v2` with:
- `test_type`: e.g., "Year 5 NAPLAN"
- `section_name`: "Writing"
- `sub_skill`: e.g., "Narrative Writing", "Persuasive Writing"
- `question_text`: The writing prompt
- `response_type`: "text" or "extended_response"
- `max_points`: The rubric total marks (e.g., 48 for NAPLAN)
